# 블록 작성 원칙

`blocks/` 의 모든 공유 스크립트가 따르는 일반 규칙. 첫 블록(`healthcheck.sh`)에서
확립했고, 이후 블록(run_container·provision 등)도 동일하게 적용한다.

## 1. 실행 위치 중립

블록은 순수 bash + 표준 도구(curl 등)만 쓴다. SSH runner 에서든 SSM in-box 에서든
같은 스크립트가 그대로 돈다. **transport(어디서 어떻게 실행되나)는 호출자 소관**이며
블록 본문에 스며들지 않는다.

## 2. 블록은 값을 담지 않는다

- **서비스마다 다른 값은 default 없는 필수 인자**로 받는다 (누락 시 즉시 에러).
  default 를 두면 특정 서비스 값과 우연히 겹쳐(예: 헬스체크 5s x 60 = server 값)
  명시 세팅을 빠뜨려도 조용히 통과한다. piki 테스트 규약의 "stub default 는 throw"
  와 같은 정신이다.
- default 는 **서비스 정체성과 무관한 공통값**에만 허용한다 (예: curl 요청 타임아웃 5s).
- **값의 SSOT 는 호출부(서비스별 배포 설정) 한 곳**이다. 블록도, 계약 문서도 값을
  소유하지 않는다.

## 3. 계약 문서와의 관계 (층 분리)

`contracts/*.md` 는 판정 규칙의 **의도**를 설명하고, `blocks/*.sh` 는 그것을 **실행**한다.
같은 규칙이 양쪽에 나타나는 건 파생 관계라 SSOT 위반이 아니다. 위반은 **값의 중복**
(파라미터 값이 두 곳에 독립적으로 박히는 것)이며, 이는 2번 원칙이 차단한다.

## 4. 종료 코드 규약

| 코드 | 의미 |
|---|---|
| 0 | 성공 |
| 1 | 실행 실패 (재시도 소진, 대상 오류 등) |
| 2 | 인자 오류 (필수 누락, 알 수 없는 인자) |

호출자는 이 코드로 "배포를 계속/중단/롤백"을 분기하므로 블록은 이 구분을 지킨다.

## 5. 셀프 검증 가능

블록은 실제 인프라 없이 로컬에서 검증 가능해야 한다 (예: python http.server 로
성공 경로, 닫힌 포트로 실패 경로, 인자 누락으로 인자 오류 경로). 블록을 추가·변경하면
성공/실패/인자오류 세 경로를 실행해 확인한 뒤 커밋한다.
